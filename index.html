linus 服务器安装nodejs ,npm

linus 服务器安装nodejs:

登陆到Linux终端，进入/usr/local/src目录，如下：

root@ubuntu:~# cd /usr/local/src/

下载nodejs安装包：

#wget http://nodejs.org/dist/v0.10.17/node-v0.10.17.tar.gz
2，解压文件并安装 
#  tar xvf node-v0.10.17.tar.gz
#  cd node-v0.10.17
#  ./configure
# make
# make install
# cp /usr/local/bin/node /usr/sbin/
查看当前安装的Node的版本 
# node -v

linus 服务器安装npm
安装NPM：

curl http://npmjs.org/install.sh | sh
如果没有权限会安装不成功，需要加上sudo来确保权限： 
curl http://npmjs.org/install.sh | sudo sh

通过sudo npm install express -g 命令安装了express，但是在执行express -V，出现 -bash: express: command not found

在官网找到了解决方法是 express 4需要通过 下面的命令安装

npm install -g express-generator

win7安装express和创建项目

首先，我们需要安装express库。在Express3.6.x之前的版本，Express需要全局安装的，项目构建器模块是合并在Express项目中的，后来这个构建器被拆分出来，独立成为了一个项目express-generator，现在我们只需要全局安装express-generator项目就行了。

npm install -g express-generator@4


安装好express-generator包后，我们在命令行就可以使用express命令了。
express -V # 检查express的版本


接下来，我们使用express的命令，来创建项目了。

express -e nodejs-demo  # 创建项目


进入项目目录，下载依赖库，构建项目。

D:\Program Files\nodejs>cd nodejs-demo && npm install -d


启动项目:
npm start

项目启动成功，打开浏览器 http://localhost:3000，就可以看到显示的页面了。

  1. 目录结构

接下来，我们详细看一下Express4项目的结构、配置和使用。
bin, 存放启动项目的脚本文件
node_modules, 存放所有的项目依赖库。
public，静态文件(css,js,img)
routes，路由文件(MVC中的C,controller)
views，页面文件(Ejs模板)
package.json，项目依赖配置及开发者信息
app.js，应用核心配置文件

从SEO角度分析手机网站到底做成响应式的，还是做独立站

很多人都比较在乎手机站，最近野狼也是频繁被问及到这个问题，所以想要集中回答一下。
首先，肯定一点，百度目前比较支持的是独立手机站，未来可能会支持响应式手机网站的判断。这一点态度对SEO有很重要的指导意义，基本可以断定目前独立手机站还是要比响应式网站好的。（注：我们说的独立手机站并不是与网站完全独立，而是说与手机站链接不是一个链接，而是电脑，手机网址一一对应的）。
